# RAG AI Assistant

A Retrieval-Augmented Generation (RAG) chatbot that lets you query your own documents using natural language. Built with ChromaDB, HuggingFace embeddings, Groq LLM, and Streamlit.

---

## How it works

The system processes your documents through four stages before answering any question:

1. **Document loading** — files from `data/` are read and parsed
2. **Chunking** — documents are split into overlapping chunks for accurate retrieval
3. **Embedding and storage** — each chunk is converted to a vector and stored in ChromaDB
4. **Retrieval and generation** — at query time, relevant chunks are fetched and passed to the LLM

---

## Tech stack

| Component | Tool |
|---|---|
| Vector database | ChromaDB |
| Embeddings | HuggingFace `sentence-transformers/all-MiniLM-L6-v2` |
| LLM | Groq API (LLaMA 3) |
| Interface | Streamlit |
| Framework | LangChain |

## Setup

### 1. Clone the repository

```bash
git clone <repo-url>
cd rag-assistant
```

### 2. Create a virtual environment

```bash
python -m venv venv
source venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Set your Groq API key

Get a free key at [console.groq.com](https://console.groq.com), then export it:

```bash
export GROQ_API_KEY="your_key_here"
```

To make it permanent:

```bash
echo 'export GROQ_API_KEY="your_key_here"' >> ~/.bashrc
source ~/.bashrc
```

### 5. Add your documents

Place `.txt` files in the `data/` folder. A sample file is included.

### 6. Run the ingestion pipeline

```bash
python ingest.py
```

This builds the vector store from your documents. Run it once, and again whenever you add new files.

### 7. Start the chatbot

```bash
streamlit run app.py
```

Open your browser at `http://localhost:8501`.

---

## Usage

1. Add documents to `data/`
2. Run `python ingest.py`
3. Run `streamlit run app.py`
4. Ask questions in natural language

The assistant will only answer from the content in your documents. If information is not found, it will say so rather than guess.

---

## Requirements

- Python 3.11+
- Groq API key
- PyTorch (CPU build is sufficient)

---

## Limitations

- Responses are not streamed — the full answer appears at once
- Embeddings run on CPU, which is slower than GPU
- Adding new documents requires re-running `ingest.py`
- Only `.txt` files are supported at this time
