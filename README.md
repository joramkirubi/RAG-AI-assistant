# RAG AI Assistant

A Retrieval-Augmented Generation (RAG) chatbot that lets you query your own documents using natural language. Built with ChromaDB, HuggingFace embeddings, Groq LLM, and Streamlit.

---

## How it works

The system processes your documents through four stages before answering any question:

1. **Document loading** — files from `data/` are read and parsed
2. **Chunking** — documents are split into overlapping chunks for accurate retrieval
3. **Embedding and storage** — each chunk is converted to a vector and stored in ChromaDB
4. **Retrieval and generation** — at query time, relevant chunks are fetched and passed to the LLM

---

## Tech stack

| Component | Tool |
|---|---|
| Vector database | ChromaDB |
| Embeddings | HuggingFace `sentence-transformers/all-MiniLM-L6-v2` |
| LLM | Groq API (LLaMA 3) |
| Interface | Streamlit |
| Framework | LangChain |

## Prerequisites

Before you begin, make sure the following are installed on your machine.

### Python 3.11+

**Windows**
Download and run the installer from [python.org](https://www.python.org/downloads/). During installation, check the box that says **"Add Python to PATH"**.

**macOS**
```bash
brew install python@3.11
```
If you do not have Homebrew: `/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"`

**Linux (Ubuntu / Debian / Kali)**
```bash
sudo apt update && sudo apt install python3.11 python3.11-venv python3-pip -y
```

Verify your installation:
```bash
python --version
# or
python3 --version
```

### Git

**Windows** — Download from [git-scm.com](https://git-scm.com/download/win)

**macOS** — `brew install git`

**Linux** — `sudo apt install git -y`

### PyTorch (CPU)

PyTorch is required by the HuggingFace embedding model. Install the CPU-only version — it is smaller and works on all machines without a GPU.

**All platforms:**
```bash
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cpu
```

> If you have an NVIDIA GPU and want faster embeddings, visit [pytorch.org](https://pytorch.org/get-started/locally/) to get the correct CUDA install command for your system.

## Setup

### 1. Clone the repository

```bash
git clone <your-repo-url>
cd rag-assistant
```

### 2. Create a virtual environment

A virtual environment keeps this project's dependencies isolated from the rest of your system.

**macOS / Linux**
```bash
python3 -m venv venv
source venv/bin/activate
```

**Windows (Command Prompt)**
```cmd
python -m venv venv
venv\Scripts\activate
```

**Windows (PowerShell)**
```powershell
python -m venv venv
.\venv\Scripts\Activate.ps1
```

You should see `(venv)` appear at the start of your terminal prompt. Run all following commands with the environment active.

### 3. Install dependencies

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

> **Linux users:** If you get a system package conflict error, add `--break-system-packages` to the pip command.

If any package fails to install individually, you can install it directly:
```bash
pip install langchain langchain-community langchain-groq langchain-huggingface
pip install chromadb sentence-transformers streamlit python-dotenv
```

### 4. Set up your environment variables

This project uses a `.env` file to store your API key. This keeps secrets out of your code and out of version control.

Copy the example file:

**macOS / Linux**
```bash
cp .env.example .env
```

**Windows**
```cmd
copy .env.example .env
```

Open `.env` in any text editor and fill in your key:

```
GROQ_API_KEY=your_actual_key_here
```

Get a free Groq API key at [console.groq.com](https://console.groq.com) — sign up, go to **API Keys**, and click **Create API Key**.

> Never commit your `.env` file. It is already listed in `.gitignore`.

### 5. Add your documents

Place `.txt` files in the `data/` folder. A sample file covering AI and RAG concepts is already included to help you test the system.

### 6. Run the ingestion pipeline

```bash
python ingest.py
```

This reads your documents, splits them into chunks, generates embeddings, and saves everything to the `vectorstore/` folder. Run it once at the start, and again any time you add or update documents.

### 7. Start the chatbot

```bash
streamlit run app.py
```

Open your browser and go to `http://localhost:8501`. The sidebar will show a green tick confirming your API key is loaded.

## Usage

1. Add `.txt` documents to `data/`
2. Run `python ingest.py`
3. Run `streamlit run app.py`
4. Ask questions in natural language

The assistant answers only from the content in your documents. If the information is not found, it will say so rather than guess.

## Troubleshooting

**`GROQ_API_KEY not set` error**
Make sure your `.env` file exists in the project root and contains the key. Double-check there are no quotes around the value — it should be `GROQ_API_KEY=gsk_abc...`, not `GROQ_API_KEY="gsk_abc..."`.

**`Vector store not found` error**
You need to run `python ingest.py` before starting the app.

**`pip` not found on Linux/macOS**
Try `pip3` instead of `pip`, or run `python3 -m pip install ...`.

**PowerShell says "running scripts is disabled"**
Run this once in PowerShell as Administrator:
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

**Slow first run**
The HuggingFace embedding model (~90MB) is downloaded on first use. Subsequent runs use the cached version.

---

## Limitations

- Responses are not streamed — the full answer appears at once
- Embeddings run on CPU, which is slower than GPU
- Adding new documents requires re-running `ingest.py`
- Only `.txt` files are supported at this time
