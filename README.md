# RAG AI Assistant

A Retrieval-Augmented Generation (RAG) chatbot that lets you query your own documents using natural language. Every answer shows exactly which document it was sourced from.

Built with ChromaDB, HuggingFace embeddings, Groq LLM, LangChain, and Streamlit.

---

## How it works

1. **Document loading** — `.txt` and `.json` files from `data/` are loaded and parsed
2. **Chunking** — documents are split into overlapping chunks for accurate retrieval
3. **Embedding and storage** — each chunk is converted to a vector and stored in ChromaDB
4. **Retrieval and generation** — at query time, the top-4 relevant chunks are fetched and passed to the LLM
5. **Source attribution** — every answer shows which documents were referenced

## Prerequisites

### Python 3.11+

**Windows** — Download from [python.org](https://www.python.org/downloads/). Check "Add Python to PATH" during install.

**macOS** — `brew install python@3.11`

**Linux (Ubuntu / Debian / Kali)** — `sudo apt update && sudo apt install python3.11 python3.11-venv python3-pip -y`

### Git

**Windows** — [git-scm.com](https://git-scm.com/download/win)  
**macOS** — `brew install git`  
**Linux** — `sudo apt install git -y`

### PyTorch (CPU build — install before requirements.txt)

```bash
pip install torch --index-url https://download.pytorch.org/whl/cpu
```

## Setup

### 1. Clone the repository

```bash
git clone https://github.com/joramkirubi/RAG-AI-assistant.git
cd RAG-AI-assistant
```

### 2. Create a virtual environment

**macOS / Linux**
```bash
python3 -m venv venv
source venv/bin/activate
```

**Windows (Command Prompt)**
```cmd
python -m venv venv
venv\Scripts\activate
```

**Windows (PowerShell)**
```powershell
python -m venv venv
.\venv\Scripts\Activate.ps1
```

You should see `(venv)` at the start of your prompt. Keep it active for all commands below.

### 3. Install dependencies

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

> **Linux users:** If you get a system conflict error, add `--break-system-packages`

### 4. Set up your API key

Copy the example file:

**macOS / Linux**
```bash
cp .env.example .env
```

**Windows**
```cmd
copy .env.example .env
```

Open `.env` and add your key:
```
GROQ_API_KEY=your_actual_key_here
```

Get a free key at [console.groq.com](https://console.groq.com) → API Keys → Create API Key.

> Never commit `.env` — it is already in `.gitignore`.

### 5. Add your documents

Place `.txt` or `.json` files in the `data/` folder. Sample files are already included.

### 6. Run ingestion

```bash
python ingest.py
```

This reads all documents, splits them into chunks, embeds them, and saves to `vectorstore/`.  
Re-run this every time you add or change documents.

### 7. Start the chatbot

```bash
streamlit run app.py
```

Open your browser at `http://localhost:8501`.

## Usage

1. Add `.txt` or `.json` files to `data/`
2. Run `python ingest.py`
3. Run `streamlit run app.py`
4. Ask questions in natural language
5. Each answer will show the source documents it was pulled from

## Supported file types

| Format | How it is loaded |
|---|---|
| `.txt` | Read directly with TextLoader, UTF-8 encoding |
| `.json` | Parsed as array or object; each item becomes a document chunk |

## Troubleshooting

**`GROQ_API_KEY not set`** — Make sure `.env` exists and contains `GROQ_API_KEY=gsk_...` with no quotes around the value.

**`Vector store not found`** — You need to run `python ingest.py` before starting the app.

**`pip not found`** — Try `pip3` or `python3 -m pip install ...`

**PowerShell execution policy error** — Run as Administrator: `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser`

**Slow first run** — The HuggingFace model (~90MB) downloads once on first use. Subsequent runs use the cached version.

---

## Limitations

- Only `.txt` and `.json` files are supported (PDF and DOCX can be added via LangChain loaders)
- No conversation memory — each question is answered independently
- Responses are not streamed
- Embeddings run on CPU
