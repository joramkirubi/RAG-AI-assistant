# MedAssist — Medical AI Assistant

A domain-specific Retrieval-Augmented Generation (RAG) chatbot for healthcare and medical question answering. Every answer is grounded in the provided medical documents, cites its sources, and is guided by a ReAct reasoning strategy.

Built with ChromaDB, HuggingFace embeddings, Groq LLM, LangChain, and Streamlit.

> **Medical Disclaimer:** This assistant provides general health information only. It is not a substitute for professional medical advice, diagnosis, or treatment. Always consult a qualified healthcare provider for personal medical decisions.

---

## Domain

This assistant is scoped to **healthcare and medical topics** including:

- Common conditions and symptoms (hypertension, diabetes, asthma, cardiovascular disease)
- Medications and drug classes (antibiotics, analgesics, antihypertensives)
- Preventive care and screening guidelines
- Mental health (depression, anxiety)
- First aid and emergency care
- Nutrition, lifestyle, and physical activity guidelines

To use it for a different domain, replace the files in `data/` and re-run `python ingest.py`.

---

## How it works

1. **Document loading** — `.txt` and `.json` files from `data/` are loaded and parsed
2. **Chunking** — documents are split into overlapping chunks for accurate retrieval
3. **Embedding and storage** — each chunk is converted to a vector and stored in ChromaDB
4. **Retrieval** — the top-5 most semantically similar chunks are fetched for each query
5. **ReAct reasoning** — the LLM reasons through the retrieved context before answering
6. **Source attribution** — every answer shows which documents were referenced
7. **Retrieval metrics** — coverage score, chunk count, and source diversity are tracked per query

## Setup

### 1. Clone the repository

```bash
git clone https://github.com/joramkirubi/RAG-AI-assistant.git
cd RAG-AI-assistant
```

### 2. Create a virtual environment

**macOS / Linux**
```bash
python3 -m venv venv
source venv/bin/activate
```

**Windows (Command Prompt)**
```cmd
python -m venv venv
venv\Scripts\activate
```

**Windows (PowerShell)**
```powershell
python -m venv venv
.\venv\Scripts\Activate.ps1
```

You should see `(venv)` at the start of your prompt.

### 3. Install dependencies

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

> **Linux users:** If you get a system conflict error, add `--break-system-packages`

### 4. Set up your API key

```bash
cp .env.example .env     # macOS / Linux
copy .env.example .env   # Windows
```

Open `.env` and add your key:
```
GROQ_API_KEY=your_actual_key_here
```

Get a free key at [console.groq.com](https://console.groq.com).

### 5. Run ingestion

```bash
python ingest.py
```

Re-run this every time you add or change documents in `data/`.

### 6. Start the chatbot

```bash
streamlit run app.py

# If the above fails due to path conflicts (common on Linux):
python -m streamlit run app.py
```

Open your browser at `http://localhost:8501`.

## Features

### Conversation memory
The assistant remembers the last 3 exchanges (6 messages) in a session, allowing natural follow-up questions such as "Can you explain that further?" or "What are the side effects of that medication?"

### Source attribution
Every answer shows a "Referenced documents" bar with the exact filename and file type used to generate the response.

### Retrieval evaluation metrics
The sidebar displays metrics after each query:

| Metric | Description |
|---|---|
| Chunks | Number of document chunks retrieved |
| Sources | Number of unique source files used |
| Avg Length | Average character length of retrieved chunks |
| Coverage | Ratio of unique sources to chunks (higher = less redundancy) |

### ReAct reasoning
The LLM internally follows: **Thought → Observe → Reason → Act** before writing its answer, producing more accurate and contextually aware responses.

## Troubleshooting

**`GROQ_API_KEY not set`** — Make sure `.env` exists with `GROQ_API_KEY=gsk_...` (no quotes around the value).

**`Vector store not found`** — Run `python ingest.py` before starting the app.

**`pip not found`** — Try `pip3` or `python3 -m pip install ...`

**PowerShell execution policy error** — Run as Administrator: `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser`

**Slow first startup** — The HuggingFace embedding model (~90MB) downloads once on first use.
